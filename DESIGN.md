# Design System — Solarheld Dealroom

> Always read this file before making any visual or UI decision. Do not deviate without explicit approval.

---

## Product Context

- **What this is:** Personalized digital offer pages (Dealrooms) for Solarheld, a German residential & commercial solar provider. Each Dealroom is a 1:1 sales surface generated by the admin tool from customer data, with embedded calculator, video, PandaDoc-signature, references, and email-flow automation.
- **Who it's for:** German B2C (Eigenheim, Hausbesitzer 35-65) und B2B (Gewerbekunden — Handwerk, Hotellerie, KMU). Sales-Lead schickt den Link; Customer öffnet auf Mobile oder Desktop und entscheidet.
- **Industry:** Residential & commercial solar in DE/EU. Concentrated, competitive, dominated by old-template websites and generic SaaS.
- **Project type:** Two surfaces in one app — (a) Public marketing/sales surface (`/d/[slug]`), (b) Admin product UI (`/dashboard/*`).

## The Memorable Thing

**"Das wirkt klar & einfach — ich verstehe sofort was ich kriege."**

Every design decision serves this. Trust + comprehension > decoration + cleverness. AIDA-narrative on the public surface is paced for clarity, not dramatic-tension. Admin surface is calm and dense like Linear, not flashy like Notion's marketing pages.

## Aesthetic Direction

- **Direction:** **Swiss Modernist Clarity** with atmospheric depth where it earns its place. Composition-first, type-led, restrained color. Anti-thesis to the 100 other solar providers that look like 2018 WordPress.
- **Decoration level:** **minimal-intentional** — type and whitespace do most of the work. Photography and atmospheric brand-color glow appear deliberately, never as background-filler.
- **Mood:** Confident, precise, calm. Modern technical premium. The product knows what it's offering; the user feels handled by professionals.
- **References:** Linear (app + marketing), Vercel marketing, Stripe Press, Apple Pro pages, Patagonia editorial, NYT interactive features.

## Typography

**Display + Body — one-family Swiss-modernist system. Strict geometric, no editorial-serif softness.**

### Primary recommendation (paid license)
- **Display (H1/H2/Hero — oversized):** **PP Neue Montreal** Medium 500 / Bold 700. Pangram. Used by Linear marketing, Vercel.
- **Body / UI / labels:** **PP Neue Montreal** Book 400 / Medium 500. Same family. One-family monosystem maximizes "klar & einfach" signal.
- **Mono / Data / Tabular:** **JetBrains Mono** 400/500 (free). For Calculator numbers, Analytics charts, Email logs, code-like data. `font-variant-numeric: tabular-nums` everywhere numbers align.
- **Licensing:** PP Neue Montreal is paid for production (~€600/yr small-business). Free for development & evaluation. Decision needed before prod-ship.

### Free fallback (if licensing is a no-go)
- **Display:** **Geist** Bold 700 / Black 900 (Vercel, MIT). Geometric, distinct from Inter family, variable font.
- **Body:** **Geist** Regular 400 / Medium 500. Same family monosystem.
- **Mono:** **Geist Mono** 400/500 (Vercel, MIT).
- **Note:** Geist is rising in popularity — convergence-risk in 2 years. Mitigate with heavy weight contrast (oversized Black display vs quiet Regular body) and tight letter-spacing on display.

### Banned (do not use as primary, even if user pastes them in)
Inter, Roboto, Arial, Helvetica, Open Sans, Lato, Montserrat, Poppins, Space Grotesk, system-ui, -apple-system. Plus Jakarta Sans (currently in use) gets retired during the token-migration.

### Loading
`next/font` with `display: swap`, subset `latin` + `latin-ext` (German `ä/ö/ü/ß/€`). Variable-font files only. `font-display: swap` to prevent FOIT.

### Type Scale (modular, ratio 1.250 perfect-fourth)

```
display-2xl  72px / 76px / -0.04em  / 600    — hero only, max 1 per page
display-xl   60px / 64px / -0.03em  / 600    — section-opener H1
display      48px / 52px / -0.025em / 500    — primary section H2
h1           36px / 40px / -0.02em  / 500
h2           28px / 32px / -0.015em / 500
h3           22px / 28px / -0.01em  / 500
body-lg      18px / 28px / -0.005em / 400    — lead paragraphs, hero subtitle
body         16px / 24px / 0        / 400    — default body
body-sm      14px / 20px / 0        / 400    — secondary text, captions
micro        12px / 16px / 0.02em   / 500    — labels, badges, uppercase tracking
```

- Display sizes use **clamp()** for fluid scaling between 320px and 1440px viewport.
- Max line-length on body copy: **65ch**. Never exceed 6 lines in a single paragraph wrap on display.
- Tracking on display (`-0.04em` to `-0.02em`) is tight on purpose — geometric sans needs tight set to feel premium.

## Color

**Approach: restrained.** One brand accent + warm stone-neutrals + 3 semantic colors + 2 supporting accents. No purple gradients, no random pastels, no 7-color UI-kit slop.

### Light Mode

```

--brand-500:  #E97E1C    /* primary CTAs, focus rings, key accents */
--brand-600:  #C66514    /* hover state on brand */
--brand-glow: rgba(233, 126, 28, 0.12)  /* atmospheric glow on hero / dark sections */

/* Foundation — warm stone, NOT cool slate */
--bg:           #fafaf9   /* page background, paper-warm */
--surface:      #ffffff   /* cards, dialogs */
--surface-sub:  #f5f5f4   /* subtle alternation, hover */

--border-strong:#d6d3d1   /* emphasized border */
```

### Color Discipline Rules
- **Brand-orange usage rule:** max **1 primary CTA + 1 accent element + atmospheric glow** per section. Currently the codebase uses `${brandColor}05/12/15/25/30` in 40+ tinted-background places — these get audited and removed during the migration.
- **No gradients** as primary visual element. Single solid colors. Exception: hero atmospheric glow (radial gradient `brand-glow → transparent`).
- **Semantic colors** for state communication only (success/warning/danger/info). Never decorative.

## Spacing

- **Base unit:** 4px
- **Scale:** `2xs(2) xs(4) sm(8) md(12) lg(16) xl(24) 2xl(32) 3xl(48) 4xl(64) 5xl(96) 6xl(128) 7xl(160)`
- **Density modes (admin only):** `compact` / `comfortable` (default) / `spacious` — exposed via CSS variable `--density` and Settings toggle.

### Semantic Spacing Tokens
```
--space-section-y-public:    clamp(64px, 12vw, 160px)   /* generous, editorial-magazine rhythm */
--space-section-y-admin:     clamp(24px, 4vw, 48px)     /* compact, product-UI */
--space-card-x:              16px
--space-card-y:              16px
--space-field-y:             8px
--space-field-stack:         12px
--space-inline-sm:           8px
--space-inline-md:           16px
```

- Max content width Public: `1200px` (hero can be full-bleed)
- Max content width Admin: `1440px` with sidebar

## Layout

- **Public Dealroom:** creative-editorial. Bento, asymmetric Compositions per AIDA stage, varied per section (no `headline → 3 cards` Repetition).
- **Admin Dashboard:** grid-disciplined. 12-col on desktop with sidebar (240px), 4-col on tablet, 1-col on mobile. Tables-first for data-dense surfaces.
- **Border radius scale:**
  ```
  --radius-xs:   4px    /* badges, micro-pills */
  --radius-sm:   6px    /* inputs, buttons */
  --radius-md:   8px    /* cards, dialogs */
  --radius-lg:   12px   /* feature cards, panels */
  --radius-xl:   20px   /* hero containers, large surfaces */
  --radius-full: 9999px /* avatars, full-pills */
  ```
- **Photography & assets:** real solar-installation photography (next/image AVIF/WebP), abstract energy visuals, textured paper backgrounds. Generated via `/imagegen-frontend-web`. Emojis (☀️⚡🏆) currently used on product-tier cards are removed.

## Motion

- **Approach:** intentional. Functional micro-motion in admin (Emil-quality), cinematic moments in public (GSAP-driven where it earns the cost).
- **Easings:**
  ```
  --ease-enter:        cubic-bezier(0.16, 1, 0.3, 1)    /* out-expo, decelerating */
  --ease-exit:         cubic-bezier(0.7, 0, 0.84, 0)    /* in-expo, accelerating */
  --ease-standard:     cubic-bezier(0.65, 0, 0.35, 1)   /* in-out, transitions */
  --ease-atmospheric:  cubic-bezier(0.83, 0, 0.17, 1)   /* slow, atmospheric reveals */
  ```
- **Durations:**
  ```
  --duration-instant:      0ms
  --duration-micro:        100ms   /* hover, focus, button-press */
  --duration-fast:         200ms   /* small UI transitions */
  --duration-base:         400ms   /* default for entrance, modal open */
  --duration-slow:         700ms   /* section transitions, scroll-reveals */
  --duration-atmospheric:  1500ms  /* hero scroll-pin, parallax depth */
  ```
- **Library split:**
  - **framer-motion** (already installed) → Component-level state, list re-orders, layout-shifts, AnimatePresence.
  - **GSAP + ScrollTrigger** (new) → Hero scroll-pinning, outcome-section parallax, calculator slider tick. Lazy-loaded on Public dealroom only — admin never imports GSAP.
- **Reduced-motion:** all GSAP scroll-pinning + parallax respect `prefers-reduced-motion: reduce`. Fall back to instant or `fast` durations.

## Elevation (Shadows)

Five-step semantic scale. Replaces ad-hoc `shadow-sm` usage everywhere.

```
--shadow-flat:     none
--shadow-raised:   0 1px 2px rgba(28,25,23,.04), 0 1px 1px rgba(28,25,23,.06)
--shadow-floating: 0 4px 12px rgba(28,25,23,.06), 0 2px 4px rgba(28,25,23,.04)
--shadow-popover:  0 12px 32px rgba(28,25,23,.08), 0 4px 8px rgba(28,25,23,.06)
--shadow-modal:    0 32px 64px rgba(28,25,23,.12), 0 12px 24px rgba(28,25,23,.08)
```

Dark mode uses same scale but with `rgba(0,0,0,.4-.8)` for visibility on dark surfaces.

## Composition Principles (Public Dealroom)

1. **One hero per page, max.** Display-2xl gets ONE use.
2. **No `headline → 3 cards` repeats.** Each section gets a distinct composition (bento, 2-col editorial, full-bleed image, sticky-scroll narrative, asymmetric grid).
3. **Photography earns its place.** No decorative stock-style photos. Every image either: shows the actual product (solar installation), supports the section's argument (energy abstract), or sets atmospheric anchor for the section.
4. **Whitespace is the primary design tool.** Generous section-y (96-160px), max 65ch body width, oversized display type with tight tracking.
5. **AIDA narrative pace:** Attention (hero) → Interest (pain visualisation) → Desire (outcome cinematic) → Action (packages + calculator + CTA). Each stage gets its own visual treatment.

## Composition Principles (Admin Dashboard)

1. **Tables for data, cards for entities.** DealroomList, Customers, EmailLogs become real tables. Templates, References stay as cards.
2. **Density modes available.** Default comfortable, power users can switch to compact.
3. **Skeleton screens, never spinners.** Loading states match layout exactly.
4. **Empty states are recommendations.** Not "Noch keine X" + dashed border. Instead: "Lege dein erstes Angebot in 3 Minuten an" + Primary-CTA inline + 1-line value-prop.
5. **Inline validation, not toasts for field errors.** Toasts only for system-level events (saved, deleted, sent). Field errors appear inline in red under the field.
6. **Optimistic UI for save-actions.** Show success state immediately, reconcile on error.

## Anti-Patterns (NEVER do these)

- Purple/violet gradients
- 3-column icon-circle grids
- Centered-everything with uniform spacing  
- Uniform bubble border-radius on all elements
- Gradient buttons as primary CTA
- Generic stock-photo hero sections
- Inter / Roboto / system-ui as primary type
- "Built for X" / "Designed for Y" marketing copy patterns
- Tinted brand-color backgrounds in 40+ places (current state)
- Plus Jakarta Sans body type (retired during migration)
- Spinners for any loading state
- Dashed-border empty states with "Noch keine X" placeholder copy
